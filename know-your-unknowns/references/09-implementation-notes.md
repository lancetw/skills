# 09 — Implementation notes 實作札記

**時機**：正在執行已核可的計畫。維護一份即時日誌，記下**每一處程式碼逼著計畫轉彎的地方**——地圖與疆域的落差在實作中才現形，事後憑記憶重建會漏。

## 做法（實作期間）

在專案根目錄維護 `implementation-notes.md`，**在驚訝的當下**追加時間戳記條目，不要事後補寫。條目分四型：

- **照計畫**——實際狀況與預期相符（也要記，證明掃過了）。
- **發現**——關於既有系統的新認識，不影響計畫。
- **偏離**——計畫與現實有落差，四欄：**計畫怎麼說**／**程式碼揭露什麼**／**當下的保守選擇**／**之後回頭看什麼**。
- **待人決定**——屬於產品判斷、不該由 agent 擅自決定的事。

## Artifact 結構（檢查點或收工時，從日誌渲染）

1. **標頭**：日誌檔路徑、時間範圍、分支＋統計（N 條 · N 偏離 · N 待你決定）。
2. **篩選頁籤**：**全部**／**偏離**／**待你決定**。
3. **條目流**：時間序，每條標類型徽章；偏離條目完整呈現四欄。
4. **頁尾「折回計畫」**：2–4 條要併回計畫的教訓（可複製）——下一輪的地圖因此更準。

## 回覆組裝

「待你決定」條目是主要輸出：使用者逐條回覆決定。「折回計畫」條列供使用者複製進計畫文件。

## 完成判準

偏離條目四欄齊全；沒有任何一條是事後從記憶重建的（時間戳記與 commit 時間對得上）。
